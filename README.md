# 안녕하세요, 장호정입니다

반복되는 화면 로직과 복잡성을 추상화로 정리해 팀 전체의 속도를 끌어올리는 프론트엔드 엔지니어입니다. 사용자가 마주하는 문제를 끝까지 풀기 위해 필요하면 컴퓨터 비전, ML 파이프라인, 인프라까지 들어가지만, 주력과 성장의 축은 프론트엔드 시스템 설계에 둡니다.

## 어떻게 일하는가

- 같은 화면 패턴이 반복되면 공통 컴포넌트와 메타데이터 기반 파이프라인으로 묶어, 도메인이 늘어도 추가 비용이 줄어들게 만듭니다.
- 기술 선택, 동작 구조 설계, 경계값 검증은 직접 합니다. 규칙은 회의가 아니라 훅과 테스트로 강제해 기본값으로 굳힙니다.
- 한 영역을 맡으면 끝까지 가져갑니다. FE가 주력이지만 필요하면 BE 계약, ML 파이프라인, 배포까지 들어가 문제를 마무리합니다.

## 기술

**주력**
TypeScript, React, Next.js (App Router), TanStack Query, React Hook Form + Zod, Zustand, Tailwind, Storybook, MSW, vitest

**그 외 경험 (협업과 운영에서 다뤄봄)**
SQL (PostgreSQL), Docker, AWS (EC2/S3/CodeDeploy), GitHub Actions

## 대표 작업

- **AI 메타데이터 중심 ERP** : 기업별로 다른 입력 정책, 검증, 자동 계산을 코드 수정 없이 메타데이터로 관리하는 ERP의 FE 파트를 총괄. 도메인마다 복제되던 폼 로직을 단계 분리 파이프라인과 공통 컴포넌트로 묶어, 메타데이터 한 객체 변경만으로 여러 도메인이 일관 동작하게 설계.
- **[AI 기반 KOSPI 200 매매 신호 서비스](https://github.com/03hoho03/sallaemallae)** : 단기 스프린트에서 FE와 AI를 겸직. 응답 계약을 먼저 타입으로 설계하고 MSW로 백엔드 의존성을 끊어 FE를 병렬로 진행했고, 매매 신호 ML 파이프라인의 수집부터 운영까지 담당.
- **골격 인식 기반 의류 추천 서비스** : WebRTC와 MediaPipe로 실시간 자세 가이드를 제공하면서, 표시용과 업로드용 캔버스를 분리해 AI에 보낼 이미지의 품질을 FE 단계에서 책임지는 촬영 파이프라인을 구축.
- **동아리 팀 빌딩 플랫폼 WTNT** : 서비스 FE와 어드민 FE 두 레포 전체와 배포 파이프라인을 직접 구축해 실서비스로 운영.
